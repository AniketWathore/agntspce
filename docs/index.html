AgntSpce

An open source desktop workspace for running and monitoring AI coding agents inside PTY-backed terminals.

AgntSpce combines workspace management, agent launches, live terminal output, git-aware status tracking, and token-compression metrics in one app. It runs entirely locally — no cloud dependency, no account required.

Requirements
  • macOS (Apple Silicon or Intel) — Windows and Linux support planned
  • Node.js 18+ and npm
  • Git installed
  • One or more AI coding CLI tools (Claude Code, Opencode, Codex, Gemini, etc.)

Setup Guide

Install AgntSpce and get it running on your machine in minutes.

Install dependencies

npm install

Run in development

npm run dev

This starts the Vite dev server and the Electron app. The server runs on 127.0.0.1:9460.

Build for production

npm run build

Package as Electron app

npm run electron:build

Outputs .dmg (mac), .exe (win), or .AppImage (linux) depending on your platform.

Preview built app

npm run electron:preview

Your First Workspace

A workspace is where all your agent sessions run. It points to a local directory or git repository.

1. Open the Create Workspace dialog

Click the Create Workspace button in the sidebar or use ⌘⇧N.

2. Choose a source

You can either:

  • Local folder — Point to any directory on your machine
  • Git URL — Clone a repository (requires gh auth)
# Or clone from the command line:
git clone https://github.com/user/repo.git ~/AgntSpce/my-project

3. Launch an agent

With your workspace active, click the + button in the terminal area or use ⌘⇧A to launch an AI agent.

4. Start working

Your agent terminal is ready. You can also open a shell terminal (⌘⇧S), browse files in the sidebar, or open the code editor.

Workspaces

Workspaces are the foundation of AgntSpce. Each workspace has its own set of sessions, git context, and file explorer.

Creating a workspace

Click Create Workspace in the sidebar, or use the command palette (⌘K). You can create from:

  • Local directory — picks any folder on disk
  • Git URL — clones the repo to ~/AgntSpce/<name>

Switching workspaces

Click a workspace in the sidebar to switch. All your sessions are preserved per workspace and restored when you switch back. The app runs setup and teardown scripts on switch if configured.

Managing workspaces

  • Rename — Edit the name in the sidebar
  • Duplicate — File > Duplicate Workspace
  • Export/Import — File > Save As / Load. Produces .workspace files
  • Delete — Moves to trash (restorable)
  • Permanent delete — Removes from trash permanently

Workspace config

Each workspace stores metadata locally in ~/.agent-workspace/workspaces/<id>.json:

  • Repository path and type
  • Worktree configuration
  • Environment variables
  • Setup and teardown scripts
  • Git URL (for cloned repos)

Launching an agent

Click the + button in the terminal area or press ⌘⇧A. A picker shows all available agents. After selecting one, a modal lets you configure:

  • Mode — Fresh (new session), Continue (resume conversation), Resume (restore interrupted)
  • Flags — Agent-specific options like YOLO mode, verbose output, debug logging
  • Model — For agents that support model selection (Gemini, Codex, Cursor Agent)
  • Reasoning effort — low/medium/high (Codex, Claude)
  • Verbosity — low/medium/high (Codex, Claude)

Once launched, the agent appears as a tab in the terminal area with its own PTY-backed session.

Status indicators

Each terminal shows its current status:

  • Idle — Waiting for input, or shell prompt detected
  • Busy — Agent is executing a task
  • Waiting — Agent is prompting for permission or input
  • Exited — Process has terminated

Auto-restart

Claude sessions can auto-restart when they exit. Enable this in Settings. The session is recreated with a prompt telling you how to start a new Claude session.

Shell Terminals

Regular shell sessions alongside your agent terminals. Useful for running git commands, build scripts, or any terminal work.

Opening a shell

Press ⌘⇧S or click the terminal icon in the activity bar. Shell terminals are visually distinct from agent terminals.

Shell panel

Shell sessions appear in a bottom panel that can be toggled via the activity bar or the View menu. The panel height is adjustable by dragging the resize handle.

Code Editor

A built-in code editor powered by Monaco Editor (the engine behind VS Code).

Opening files

Click any file in the File Explorer sidebar. The file opens in a new tab in the editor area.

Editing

  • Multi-tab editing with dirty indicators (dot shows unsaved changes)
  • Language detection by file extension
  • Syntax highlighting for 20+ languages
  • Scroll position is preserved per file

Saving

Press ⌘S to save the current file. Changes are written to disk via the backend.

Supported languages

TypeScript, JavaScript, Python, Rust, Go, Ruby, Java, C++, C, CSS, HTML, JSON, Markdown, YAML, TOML, SQL, XML, Shell, and more.

File Explorer

Browse your workspace files directly in the sidebar.

Tree view

The file tree shows your workspace directory, sorted with directories first. Dotfiles are hidden. Click to expand/collapse folders.

File operations

  • Create file — Create an empty file in any directory
  • Create folder — Create a new subdirectory
  • Rename — Rename files and folders
  • Delete — Delete files or entire directories

All operations go through the backend and write directly to disk.

Git Integration

Stay aware of your repository state without leaving the app.

Branch tracking

Each session shows its current git branch. Branches are polled every 30 seconds and cached with a 30-second TTL.

Git Review panel

Open the Git Review panel from the activity bar. It provides:

  • Commit log — Recent commits with hash, message, author, relative date
  • Branches — All branches sorted by committerdate
  • Working tree diff — Unstaged changes
  • Commit diff — Changes in a specific commit
  • File diff — Per-file diff, including new files
  • Working tree files — Changed files with status, additions, deletions

Caveman Mode

Reduce token usage by instructing your AI agent to respond more tersely. Caveman Mode injects style rules directly into the agent's context file.

How it works

When enabled, AgntSpce writes a SKILL.md (for Opencode) or CLAUDE.md (for Claude Code) into your workspace. These files contain instructions that tell the agent to be more concise.

Three levels

Lite

~30% fewer tokens

Professional tone, no filler words (just, really, basically). Full grammar and sentences preserved.

Full

~65% fewer tokens

Fragments allowed, articles dropped, short synonyms. "Caveman" style while keeping technical terms exact.

Ultra

~75% fewer tokens

Telegraphic style, abbreviations, no conjunctions. Maximum compression for maximum savings.

Enabling Caveman Mode

Click the Caveman icon (🪨) in the activity bar to open the panel. Toggle it on per-session and select your desired level.

Disabling

Toggle off in the Caveman panel, or tell your agent "stop caveman" or "normal mode". The skill/CLAUDE.md files are cleaned up automatically.

Output Filter

The output filter reduces noise in terminal output before it reaches you, saving tokens and screen space.

What it filters

  • ANSI escape sequences — Strips color codes and control sequences
  • Duplicate lines — Deduplicates repeated output
  • Progress bars — Removes progress indicators like 12/50 24%
  • Empty lines — Collapses or strips blank lines
  • Truncation — Limits lines by count or character length
  • Skip/Keep patterns — Custom regex rules to filter specific output
  • Match-output rules — Replaces matched output with a summary

Debug panel

Open the Output Filter Debug panel from the activity bar to see real-time stats:

  • Original vs filtered bytes and tokens
  • Reduction percentage
  • Rules applied per event
  • Event history

Command tracking

The filter detects commands entered in the terminal and tracks their output separately. This powers the command history and per-command token savings.

Parallel Tasks

Launch multiple agent sessions simultaneously across different worktrees.

How it works

Parallel tasks create N agent sessions at once, each in its own worktree directory. A shared prompt is sent to all sessions after launch.

Concurrency limits

The agent orchestrator limits parallel execution to 6 concurrent sessions. Additional requests are queued. The orchestrator also monitors:

  • CPU usage per session
  • Memory usage per session (threshold: 1GB)
  • Health checks every 15 seconds
  • Restart limits (max 3 per 60-second window)

Use cases

  • Run the same task across multiple branches
  • Test different approaches in parallel
  • Process multiple independent modules simultaneously

Worktrees

Git worktrees allow you to check out multiple branches simultaneously in separate directories.

Enabling worktrees

Worktrees are configured per workspace. When enabled, AgntSpce creates worktree directories (e.g., work1, work2) alongside your repo and creates git worktrees for each.

Adding a worktree

Use the socket event add-worktree or the worktree API. A new branch (<worktreeName>-dev) is created from the base branch (master/main) and a git worktree is set up.

Removing a worktree

Related sessions are closed, then the git worktree is removed. Use the remove-worktree event.

Worktree-aware agents

Agents with supportsWorktree: true (Claude, Opencode, Codex, etc.) automatically get their own worktree directory when launched in parallel.

Configuration

config.json (project root)

{
  "server": {
    "port": 9460,
    "host": "127.0.0.1"
  },
  "sessions": {
    "timeoutMs": 1800000,
    "maxBufferSize": 100000,
    "maxProcessesPerSession": 50
  },
  "logging": {
    "level": "info"
  }
}

~/.agent-workspace/config.json (user config)

Auto-generated on first run. Controls active workspace, recent workspaces list, theme preference, and whether to remember the last workspace on restart.
